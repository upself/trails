帳戶明細說明

在 Account Detail 頁面上可選取上傳掃描資料、管理聯絡人及管理軟硬體差異，也可產生帳戶差異報告、帳戶資產報告及軟體多重報告。

帳戶表格說明

ID：這是由 Customer Number Database 產生的帳戶 ID 號碼，而每個帳戶都有唯一的 ID。按一下帳戶 ID，即可開啟 Account Detail 頁面，以檢視該帳戶的詳細資料。帳戶 ID 旁的綠色勾號表示 Customer Number Database 中作用中的帳戶；紅色 X 號則表示非作用中的帳戶。

Name：於 Customer Number Database 中列出的帳戶名稱。

Type：定義帳戶內已保留資產類型的帳戶類型。

Dept：管理帳戶的資產管理部門。所列出的資產管理部門是部門代碼或國碼。

帳戶統計數據表格說明

每個帳戶的詳細資料都列在 Account Statistics 表格中。

HW Discrepancies：在更新硬體基準線前，於帳戶中建立為位置保留元之硬體 LPAR 的差異總數。

HW LPARS：硬體基準線與硬體 LPAR 差異中的作用中硬體 LPAR 總數。

HW with scan：包含已安裝軟體資訊的作用中硬體 LPAR 總數，該資訊來自掃描或手動軟體載入。

HW w/o scan：不含任何已安裝軟體資訊的作用中硬體 LPAR 總數。

HW% w/scan：包含已安裝軟體資訊的作用中硬體 LPAR 總數百分比，該資訊來自掃描或手動軟體載入。

SW Discrepancies：軟體產品差異數，或根據軟體產品清單驗證過的軟體產品數目。

SW：作用中並已安裝軟體的產品總數。

SW LPARS w/o HW：不含任何相關硬體 LPAR 的硬體記錄總數。

LPAR 名稱/序號搜尋欄位說明

如果已知 LPAR 名稱（主機名稱）或機器序號，請將之鍵入搜尋欄位。必要的話，可使用局部字串。

複合清單表格說明

複合清單表格會列出帳戶摘要。工具依預設，將僅傳回硬體基準線中列示為 Active 的硬體記錄，或是硬體基準線中不具狀態的資產。按一下清單中的硬體名稱，可顯示 LPAR Detail 頁面。按一下 Show All（位於表格上方），即可檢視狀態為 Inactive、HWCOUNT 或 Removed（之前已從硬體基準線移除的資產）的資產。工具也會傳回作用中的軟體掃描。畫面也將不會顯示非作用中或已移除的掃描。

HW Name：機器的名稱，也可稱為主機名稱、機器名稱或節點名稱。

SW Name：機器的名稱，也可稱為主機名稱、機器名稱或節點名稱。

Acct #：由 Customer Number Database 產生的帳戶 ID 號碼，而每個帳戶都有唯一的 ID。

Type：機型，例如伺服器或工作站。

HW Serial：硬體序號。

SW Serial：軟體序號。

ATP Status：硬體基準線中的硬體部署狀態。

Software LPARs w/o Hardware

Name：機器的名稱，也可稱為主機名稱、機器名稱或節點名稱。

Account ID：由 Customer Number Database 產生的帳戶 ID 號碼，而每個帳戶都有唯一的 ID。

Bios Serial：與軟硬體的序號相同。

Hardware LPARs w/o Software 表格說明

LPAR Name：機器的名稱，也可稱為主機名稱、機器名稱或節點名稱。

Type：硬體基準線中識別之機器的資產類型。類型有 Server、Mainframe、AS400 或 Workstation。

Serial：從硬體基準線所取得的資產序號。

ATP Status：硬體基準線中的硬體部署狀態。ATP Status 可以是 Active、Inactive、HWCOUNT、Removed 或不顯示（若移入硬體基準線中）。

Hardware w/o Hardware LPAR 表格說明

Customer Number：客戶編號。

Type：硬體基準線中識別之機器的資產類型。類型有 Server、Mainframe、AS400 或 Workstation。

Serial：從硬體基準線所取得的資產序號。

Country：硬體基準線中資產的位置。

Owner：資產擁有者。

ATP Status：硬體基準線中的硬體部署狀態。ATP Status 可以是 Active、Inactive、HWCOUNT、Removed 或不顯示（若未移入硬體基準線中）。

LPAR 明細說明

硬體 LPAR 表格說明

如有硬體 LPAR，其名稱會顯示在 Hardware LPAR 表格中。

Name：機器的名額，也可稱為主機名稱、機器名稱或節點名稱。

Machine Type：硬體基準線的四位字元識別碼。

Asset Type：機器類型，如伺服器、大型主機或工作站。

Serial number：硬體基準線中的資產序號。

Country：硬體基準線中資產的位置。

Discrepancy：：用於識別已提交之資產記錄的軟體差異類型。None 指資產記錄沒有問題。Missing 指已提出差異，但該資產的硬體基準線中沒有硬體記錄。如果列出的差異為 "missing"，則需更新硬體基準線。

若沒有硬體記錄，可按一下 No Hardware Record，開啟 Create Hardware 頁面。建立硬體資產記錄的差異（位置保留元），直到可在來源系統中更新基準線為止。

軟體 LPAR 表格說明

Software LPAR 表格包含掃描結果。若表格中顯示 "No Software Records"，則表示沒有此主機名稱的軟體記錄。按一下 No Software Records 以開啟 Select Missing Software 頁面。應新增掃描資料或遺漏軟體差異至此資產的 BRAVO。

Name：機器的名稱，也可稱為主機名稱、機器名稱或節點名稱。

Processor：資產中找到的實體處理器數量。

Effective Processors：現行有效的處理器數量。

BIOS Serial：若適用，即指硬體或軟體的序號。

Scan Time：工具自動掃描資產的日期與時間。

Acquisition Time：資料檔上傳至 BRAVO 的日期與時間。

複製 LPAR 說明

可從 "FROM" LPAR 複製所有產品驗證，至 "TO" LPAR 上的每個對應產品。複製並不會改寫 "TO" LPAR 上任何已完成的產品驗證。複製功能不只可套用於產品櫃中所管理的產品，也可套用在整個 LPAR 的驗證資訊。在大型主機驗證期間複製 LPAR。

1.                  驗證 "FROM" LPAR 上的軟體產品。

2.                  開啟 Lpar Detail 頁面，在 Actions 窗格中按一下 Copy This LPAR。
畫面會顯示帳戶的 LPAR 清單。

3.                  選取一或多個 LPARs ON THE SAME CPU ，即與 FROM LPAR 位於相同 CPU 上的 LPAR。

4.                  按一下視窗底端的 Copy。
您會收到一封電子郵件，通知正在進行複製作業。

5.                  檢閱 "TO" LPAR 清單，必要的話請變更驗證。

帳戶差異說明

Account Discrepancies 報告提供已向軟體提交所有差異的帳戶清單。差異的定義如下：

NONE 表示產品尚未進行驗證。

FALSE HIT 表示 Softaudit 已識別未安裝於系統上的產品。

INVALID 表示產品已安裝，但尚未進行驗證。

附註：若產品不屬於下列 Invalid Software Categories 中的任何情況，則會標示為 Valid。

Blocked in IFAPRD 指產品在 IFAPRD 成員中列為 Disabled。

Customer managed 指客戶自行管理產品的支援與安裝。

Duplicate product - In Use - 若 SoftAudit 已多次識別該產品，便會將其標示為重複產品複的產品，並在重複出現時新增註解以指出產品名稱。請確認已標示正確出現項目為有效產品。

Misidentification - Softaudit 識別此為某項產品，但其實是另一項產品。請新增註解以指出正確的產品名稱。

Part of Another Product 表示產品是另一項產品的免費或隨附功能，無法單獨使用。請新增註解以指出屬於哪項產品。

Shared DASD - (not in use on this LPAR) 是位於 Shared DASD 上的產品，並在另一個系統上使用，但此系統並不需要。請新增註解以指出此產品用於哪個 LPAR。

Vendor Key Required but Not Present 表示產品已安裝，但因需要金鑰而無法執行，但仍未安裝金鑰。

Restrictive vendor key - 部分供應商會結合數項相關產品成一份套件，然後為部分產品組合提供特定金鑰（例如您已取得授權的產品）。雖然仍能安裝其他產品，但無法利用此有限金鑰存取，例如 SAS。請將無法存取的其他產品標示為 Invalid Restrictive Vendor Key，然後新增註解以指出相關產品套件的名稱。

Tivoli Configuration Manager (TCM) 說明

TCM Standalone Script 的處理程序，可用於執行掃描及編譯掃描檔案為 TAR 檔案，也可執行軟體資產管理掃描。如需執行 TCM 型 script 的相關資訊，請至 http://tap.raleigh.ibm.com/standalone/，並尋找「Tivoli 獨立掃描工具的程序」。或者，請至 http://tap2.raleigh.ibm.com/ 的 Asset Management 首頁，並在左導覽窗的 Software Scanning 下，按一下 Stand-alone TCM Script (Standard)。
