「アカウント詳細 （Account Detail）」ページのヘルプ

「アカウント詳細 (Account Detail)」ページでは、スキャン・データのアップロード、連絡先の管理、ソフトウェアとハードウェアの矛盾の管理を行うためのアクションを選択できます。アカウント矛盾、アカウント資産に関するレポートのほか、ソフトウェア・マルチ・レポートを生成することもできます。

「アカウント (Account)」テーブルのヘルプ

ID (ID): CNDB で生成されたアカウント ID 番号で、アカウントごとに異なります。アカウント ID をクリックすると「アカウント詳細 (Account Detail)」ページが開き、そのアカウントの詳細が表示されます。アカウント ID の隣に表示される緑色のチェック・マークは CNDB 上のアクティブなアカウントであることを示し、赤色の X マークは非アクティブなアカウントであることを示します。

名前 (Name): CNDBにリストされるアカウント名です。

タイプ (Type):アカウント内に保存される資産タイプを定義するアカウント・タイプです。

部門 (Dept): アカウントを管理している資産管理部門です。リストされる資産管理部門は、部門コードまたは国別コードです。

「アカウント統計 (Account Statistics)」テーブルのヘルプ

「アカウント統計 (Account Statistics)」テーブルには、アカウントごとの詳細がリストされます。

HW 矛盾 (HW Discrepancies):ハードウェア・ベースラインの更新前にアカウント内のプレースホルダーとして作成されたハードウェア LPAR 矛盾の数です。

HW LPAR (HW LPARS):ハードウェア・ベースライン内のアクティブなハードウェア LPAR をハードウェア LPAR 矛盾と合計した数です。

スキャンありの HW (HW with scan):スキャンまたは手動ソフトウェア・ロードによって取得したインストール済みソフトウェア情報を含むアクティブなハードウェア LPAR を合計した数です。

スキャンなしの HW (HW w/o scan):インストール済みソフトウェア情報を含まないアクティブな HW LPAR を合計した数です。

スキャンありの HW の割合 (HW% w/scan):スキャンまたは手動ソフトウェア・ロードによって取得したインストール済みソフトウェア情報を含むアクティブな HW LPAR 全体の割合です。

SW 矛盾 (SW Discrepancies):ソフトウェア製品のリストに対して確認されたソフトウェア製品またはソフトウェア製品矛盾の数です。

SW:アクティブなインストール済みソフトウェア製品を合計した数です。

HW なしの SW LPAR (SW LPARS w/o HW):ハードウェア LPAR が関連付けられていないハードウェア・レコードを合計した数です。

「LPAR 名/シリアル検索 (LPAR Name/Serial Search)」フィールドのヘルプ

LPAR 名 (ホスト名) またはマシンのシリアル番号がわかっている場合は、この検索フィールドに入力します。必要に応じて、部分的なテキスト・ストリングを入力することもできます。

「コンポジット・リスト (Composite List)」テーブルのヘルプ

「コンポジット・リスト (Composite List)」テーブルには、アカウントの要約がリストされます。デフォルトで、このツールでは、ハードウェア・ベースライン内に「Active」として表示されたハードウェア・レコードか、ハードウェア・ベースライン内にステータスが設定されていない資産のみが返されます。リスト内のハードウェア名をクリックすると、「LPAR 詳細 (Lpar Detail)」ページが表示されます。テーブルの上側にある「すべての表示 (Show All)」をクリックすると、ステータスが「Inactive」、「HWCOUNT」、または「Removed」の資産 (ハードウェア・ベースラインから資産が事前に削除されていた場合) を表示することができます。このツールでは、アクティブなソフトウェア・スキャンも返されます。非アクティブなスキャンまたは削除されたスキャンは表示されません。

HW 名 (HW Name): マシンの名前ですが、ホスト名、マシン名、またはノード名とも呼ばれます。

SW 名 (SW Name): マシンの名前ですが、ホスト名、マシン名、またはノード名とも呼ばれます。

アカウント番号 (Acct #): CNDB で生成されたアカウント ID 番号で、アカウントごとに異なります。

タイプ (Type):サーバー、ワークステーションなどのマシン・タイプです。

HW シリアル (HW Serial): ハードウェアのシリアル番号です。

SW シリアル (SW Serial): ソフトウェアのシリアル番号です。

ATP ステータス (ATP Status): ハードウェア・ベースラインから取得されたハードウェア展開ステータスです。

「ハードウェアを伴わないソフトウェア LPAR (Software LPARs w/o Hardware)」テーブルのヘルプ

名前 (Name):マシンの名前ですが、ホスト名、マシン名、またはノード名とも呼ばれます。

アカウント ID (Account ID): CNDB で生成されたアカウント ID 番号で、アカウントごとに異なります。

BIOS シリアル (Bios Serial): ハードウェアまたはソフトウェアのシリアル番号と同じです。

「ソフトウェアを伴わないハードウェア LPAR (Hardware LPARs w/o software)」テーブルのヘルプ

LPAR 名 (LPAR Name): マシンの名前ですが、ホスト名、マシン名、またはノード名とも呼ばれます。

タイプ (Type):ハードウェア・ベースライン内で特定されたマシンの資産タイプです。タイプは、サーバー、メインフレーム、AS400、またはワークステーションです。

シリアル＃ (Serial): ハードウェア・ベースラインから取得された資産のシリアル番号です。

ATP ステータス (ATP Status): ハードウェア・ベースラインから取得されたハードウェア展開ステータスです。ATP ステータスは、ACTIVE、INACTIVE、HWCOUNT、REMOVED のいずれかですが、ハードウェア・ベースラインに設定されなかったときには表示されません。

「ハードウェア LPAR を伴わないハードウェア (Hardware w/o Hardware LPAR)」テーブルのヘルプ

カスタマー番号 (Customer Number): カスタマー番号です。

タイプ (Type):ハードウェア・ベースライン内で特定されたマシンの資産タイプです。タイプは、サーバー、メインフレーム、AS400、またはワークステーションです。

シリアル＃ (Serial): ハードウェア・ベースラインから取得された資産のシリアル番号です。

国 (Country): ハードウェア・ベースラインから取得された資産の場所です。

所有者 (Owner): 資産の所有者です。

ATP ステータス (ATP Status): ハードウェア・ベースラインから取得されたハードウェア展開ステータスです。ATP ステータスは、ACTIVE、INACTIVE、HWCOUNT、REMOVED のいずれかですが、ハードウェア・ベースラインに設定されなかったときには表示されません。

「LPAR 詳細 (Lpar Detail)」ページのヘルプ

 

「ハードウェア LPAR (Hardware LPAR)」テーブルのヘルプ

ハードウェア LPAR が存在する場合は、「ハードウェア LPAR (Hardware LPAR)」テーブルに名前が表示されます。

名前 (Name):マシンの名前ですが、ホスト名、マシン名、またはノード名とも呼ばれます。

マシン・タイプ (Machine Type): ハードウェア・ベースラインから取得された 4 文字の ID です。

資産タイプ (Asset Type):サーバー、メインフレーム、ワークステーションなどのマシンです。

シリアル＃ (Serial number): ハードウェア・ベースラインから取得された資産のシリアル番号です。

国 (Country): ハードウェア・ベースラインから取得された資産の場所です。

矛盾 (Discrepancy): 資産レコードに関して送信されたハードウェア矛盾のタイプを示します。「None」は、その資産レコードに問題がないことを意味します。「Missing」は、矛盾が送信されたが、この資産のハードウェア・ベースライン内にハードウェア・レコードが存在しないことを意味します。矛盾が「missing」としてリストされている場合は、ハードウェア・ベースラインを更新する必要があります。

ハードウェア・レコードが存在しない場合は、「No Hardware Record」をクリックして「ハードウェアの作成 (Create Hardware)」ページを開くことができます。ソース・システム内のベースラインが更新可能になるまで、ハードウェア資産レコード用の矛盾 (プレースホルダー) を作成します。

「ソフトウェア LPAR (Software LPAR)」テーブルのヘルプ

「ソフトウェア LPAR (Software LPAR)」テーブルにはスキャン結果が格納されます。テーブル内に「No Software Records」と表示されている場合は、このホスト名に関するソフトウェア・レコードが存在しません。「No Software Records」をクリックして、「欠落しているソフトウェアの選択 (Select Missing Software)」ページを開きます。スキャン・データの矛盾または欠落しているソフトウェアの矛盾を、この資産の BRAVO に追加する必要があります。

名前 (Name):マシンの名前ですが、ホスト名、マシン名、またはノード名とも呼ばれます。

プロセッサー (Processors): 資産上で発見された物理的なプロセッサーの数です。

有効なプロセッサー (Effective Processors): 現時点で有効なプロセッサーの数です。

BIOS シリアル (Bios Serial): ハードウェアまたはソフトウェア (該当する方) のシリアル番号です。

スキャン時間 (Scan Time): 自動ツールが資産をスキャンした日時です。

取得時間 (Acquisition Time): データ・ファイルが BRAVO にアップロードされた日時です。

「ソフトウェア (Software)」テーブルのヘルプ

「ソフトウェア (Software)」テーブルにリストされたソフトウェアには、以下の情報が関連付けられています。

名前 (Name):スキャン中に見つかったソフトウェアの名前です。ソフトウェア名をクリックすれば、ソフトウェアに関するその他の詳細を「ソフトウェア詳細 (Software Detail)」ページに表示することができます。このページでは、ソフトウェア矛盾を更新したり、「このソフトウェアを含むすべての LPAR」レポートを入手することができます。

製造元 (Manufacturer):ソフトウェアの製造元です。

矛盾 (Discrepancy): 資産レコードに関して送信されたソフトウェア矛盾のタイプを示します。これには、None、Valid、Invalid、False Hit があります。

製造元 (Manufacturer):ソフトウェアの製造元です。

「LPAR 連絡先の作成 (Create LPAR Contacts)」のヘルプ

特定の LPAR を担当する技術担当者のブルー・ページを検索して、その連絡先の名前を LPAR 連絡先リストに追加することができます。

ドロップダウン・フィールドで、検索に使用する検索条件のタイプを選択します。

検索フィールドに、検索対象となる人物の情報を入力します。

検索の実行後、LPAR 連絡先として作成したい人物の隣にあるプラス・アイコンをクリックします。

「LPAR 連絡先の更新 (Update LPAR Contacts)」のヘルプ

ブルー・ページから情報を取得して、LPAR 連絡先に関する情報を更新することができます。

ID (ID): 連絡先の ID 番号です。

名前 (Name): 連絡先の名前です。

シリアル＃ (Serial): 連絡先のシリアル番号です。

ファースト・ライン・マネージャー (1st Line Mgr): 連絡先のファースト・ライン・マネージャーを表す番号です。

セカンド・ライン・マネージャー (2nd Line Mgr): 連絡先のセカンド・ライン・マネージャーを表す番号です。

サード・ライン・マネージャー (3rd Line Mgr): 連絡先のサード・ライン・マネージャーを表す番号です。

マネージャー識別 (Is Manager): この人物がマネージャーなら Y、そうでなければ N が表示されます。

「この LPAR をコピー (Copy This LPAR)」のヘルプ

コピー元 LPAR からの製品確認のすべてを、コピー先 LPAR の対応する各製品にコピーすることができます。コピー先の LPAR 上で完結している製品確認は、コピーによって上書きされることがありません。このコピー機能は、製品タワーの内部で管理している製品に限らず、LPAR 全体で確認を適用します。メインフレームの確認中に LPAR をコピーする手順は以下のとおりです。

1.                   コピー元 LPAR 上のソフトウェア製品を確認します。

2.                   「LPAR 詳細 (Lpar Detail)」ページの「アクション (Actions)」パネルで、「この LPAR をコピー (Copy This LPAR)」をクリックします。
そのアカウントの LPAR が一覧表示されます。

3.                   「同じ CPU 上の LPAR (LPARs ON THE SAME CPU)」をコピー元 LPAR として選択します (複数可)。

4.                   ウィンドウ下部の「コピー (Copy)」をクリックします。
コピーが進行中であることを伝える電子メールが届きます。

5.                   必要があれば、コピー先 LPAR のリストを調査して確認データを修正します。

「有効なプロセッサー数の作成 (Create Effective Processor Count)」のヘルプ

有効なプロセッサー数は、ハードウェア資産上の物理的なプロセッサーの実数です。論理的なプロセッサーを報告する自動スキャン・ツールで取得された論理的なプロセッサーの数と、物理的なプロセッサーの実際の数とが異なる場合は、プロセッサーを追加する必要があります。

有効なプロセッサー数 (Effective Processor Count): ハードウェア資産上の物理的なプロセッサーの実数です。

ステータス (Status):プロセッサーのステータスです。

「有効なプロセッサー数 (Effective Processor Count)」テーブルのヘルプ

プロセッサー数 (Processor Count): 自動スキャン・ツールで見つかったプロセッサーの数です。

ステータス (Status): プロセッサーのステータスです。

編集者 (Editor):この情報に関する変更を行う人物の情報です。

記録時間 (Record Time): この情報を編集者が最後に変更した時間です。

「ソフトウェア矛盾の作成 (Create SW Discrepancy)」のヘルプ

LPAR の下にリストされていない製品について、欠落しているソフトウェア矛盾を作成することができます。カタログ上の正しい製品名を検索することで、追加の情報 (シグニチャー・ファイル名、バイト・サイズなど) を入力し、その製品が将来のスキャンで特定されるようにすることができます。

「TLCMz ソフトウェアのアップロード (Upload TLCMz Software)」のヘルプ

TLCMz スキャン・ファイルを資産管理サーバーに直接アップロードすることができます。このファイルは、データベースにロードするためのキューに入り、アップロードが完了すると BRAVO に結果が表示されます。

「ファイル名 (Filename)」フィールドの隣にある「参照 (Browse)」をクリックして、アップロードするファイルを探します。

「ソフトウェア矛盾のアップロード (Upload Software Discrepancy)」のヘルプ

欠落しているソフトウェア矛盾の一括ロード、または自動スキャン使用不能時に手動で収集されたインストール済みソフトウェアの一括ロードに使用する標準ファイルをアップロードできます。このファイルは、データベースにロードするためのキューに入り、アップロードが完了すると BRAVO に結果が表示されます。

「ファイル名 (Filename)」フィールドの隣にある「参照 (Browse)」をクリックして、アップロードするファイルを探します。

「TCM スクリプト TAR ファイルのアップロード (Upload TCM Script TAR File)」のヘルプ

「TCM スクリプト TAR ファイルのアップロード (Upload TCM Script TAR File)」ページでは、複数の .tar.z スキャン結果ファイルから作成された .tar ファイルを、資産管理サーバーに直接アップロードすることができます。このファイルは、データベースにロードするためのキューに入り、アップロードが完了すると BRAVO に結果が表示されます。

「ファイル名 (Filename)」フィールドの隣にある「参照 (Browse)」をクリックして、アップロードするファイルを探します。

アカウント矛盾のヘルプ

アカウント矛盾レポートでは、ソフトウェアに対して送信されたすべての矛盾のアカウントに関するリストが提供されます。矛盾は以下のように定義されます。

NONE は、その製品がまだ確認されていないことを意味します。

FALSE HIT は、システムにインストールされていない製品が SoftAudit によって識別されたことを意味します。

INVALID は、ソフトウェアがインストールされているが、確認されていないことを意味します。

注:以下の無効なソフトウェア・カテゴリーに入らない製品は、「Valid」としてマークする必要があります。

Blocked in IFAPRD は、IFAPRD メンバー内で製品が「Disabled」としてリストされている場合です。

Customer managed は、お客様が製品のサポートとインストールを管理している場合です。

Duplicate product - In Use - SoftAudit が同じ製品を複数回特定した場合は、重複製品としてマークし、重複する製品名を含むコメントを追加する必要があります。正しいオカレンスが有効な製品としてマークされていることを確認します。

Misidentification - これは SoftAudit によって 1 つの製品として識別されましたが、実際は別の製品です。コメントを追加して、正しい製品名を入力します。

Part of Another Product は、その製品が別の製品のフリー機能か、別の製品の一部であり、単独で使用できないことを意味します。どの製品にその部分が属しているかに関するコメントを追加します。

Shared DASD - (not in use on this LPAR) - その製品は、共用 DASD 上に存在し、別のシステムで使用されていますが、このシステムでは必要ありません。どの LPAR 上でその部分が使用されているかに関するコメントを追加します。

Vendor Key Required but Not Present は、その製品はインストールされていますが、必要なキーがインストールされていないため実行できないということを意味します。

Restrictive vendor key - ライセンス製品のように、関連する製品を単一のパッケージに統合し、そのような製品の組み合わせに特有のキーを発行するベンダーも存在します。SAS などの他製品がインストールされていますが、この制限付きキーではアクセスすることができません。アクセスできないこれらの他製品は「Invalid」とマークします。 Restrictive vendor key - どの製品パッケージにその部分が属しているかに関するコメントを追加します。
