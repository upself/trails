What is Signature Bank?

The Signature Bank tool is your one stop shop for software knowledge base research and administration. From this Web interface you can administer the current signature repository, search the filter library, edit the software catalog, and manage signature replication to software inventory scanning infrastructures throughout IBM.

What is the software knowledge base in the Signature Bank?

The IBM SW knowledge base includes:

  • Catalog of SW manufacturers and products
  • Intellectual capital enabling automated installed SW inventories ready for asset management and other business purposes
  • Data referenced to map, filter, consolidate or otherwise convert raw SW data collected from a scanning technology into a useful SW inventory

Where did the data in Signature Bank come from?

The Global Signature Bank Team can perform updates and add information, as necessary. Signature Bank Team members also manage manufacturer, software, software category, and bank account information. Additionally they upload signature and filter files and manage reports and historical information.

Who is the Signature Bank team?

The Signature Bank Team's mission is to improve and maintain quality and completeness of the IBM knowledge base used for software (SW) detection, including SW products installed on the mainframe and distributed platforms.

How can I access Signature Bank?

There are two types of access; read and administrative access. By default, the Asset Management community has read access. If you are outside this group and need either read or administrative access, go the Asset Management PR database and submit a service request ticket to the Signature bank tool. Include your e-mail address in the format, IBMer@us.ibm.com in the request.

What is a Signature Bank Account?

Bank accounts represent software inventory scanning infrastructure which are used as a source of software inventory data.

What is a Signature?

A signature is a specific file name and file size that is linked to a product in the signature bank catalog. This file, if found installed on a system, identifies that a particular software product and version is installed. The best option for a signature is an executable file installed that initiates a software product.

For example,

Ex SW Signature File

File Name (key exe file)ex. EXCEED.EXE
File Size in Bytesex. 5337088
Software Product Nameex. Hummingbird Exceed
Software Versionex. 2.1

What is a filter?

A filter is a product description and version that is linked to a product in the signature bank catalog. A filter is matched against the registry information of a system after a scan has been completed to identify if software is installed.

What is the difference between a manufacturer and a vendor in Signature Bank?

The manufacturer is the company that created the application's code. The vendor is the name of the company that sells the application. There may also be multiple vendors for any application. Signature Bank asks for manufacturer names as opposed to vendor names, so be sure you type manufacturer names into the appropriate fields in the tool.

How long does it take for changes I make to reflect in Signature Bank?

Changes you make in Signature Bank are immediate. Changes made are also reflected in the BRAVO application after a full reload has occurred (usually a daily occurrence).

What is a software category?

A software category (hierarchy) was created in Signature Bank to identify multiple products that are separate versions of each other (for example, Microsoft SQL Advanced Edition and Microsoft SQL Standard Edition) and give them a priority. The tools reporting installed software such as TRAILS or the Microsoft Wizard only report on the highest priority product in the list, if multiple products are discovered in a scan.

Code is often reused from version to version by manufacturers of software. What can happen when code is reused is that the scan picks up the current version as well as the lower version of the software as installed. In some cases the lower and higher versions of a product fall under different names and would show up as two line items in a scan report. This can cause a licensing issue. Only the higher of the two versions would need to be licensed and thus only the higher should be reported as installed.

What is the Software License Level field in Signature Bank? What is a software component? What is a software bundle?

This field is where you indicate whether the software you want to add is licensable, un-licensable, or a component of another family of software. Generally, if you can purchase a product, then it is licensable. Unlicensable software is freeware or any other software packages that do not require a procured license. A component is a software package which comes as part of a licensable software product. A component would never be installed apart from its parent, licensable software product; and would never require its own procured license. The component is similar, but not identical to, a software bundle. A software bundle is when a licensable software product comes free with another licensable software product; but would not be free if it were installed as a standalone product.

Can Signature Bank data be distributed externally?

No. Data should not be shared with anyone outside of IBM, including software manufacturers, vendors, and customers.
